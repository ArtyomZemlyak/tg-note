## Работа с медиафайлами

Папка `media/` теперь хранит **все типы вложений**:

- `img_*.jpg|png|webp` — изображения (+ `.md` и `.json` с описанием/OCR)
- `vid_*.mp4`, `screen_*.mov`, `deck_*.pdf` — видео, скринкасты, презентации
- `aud_*.mp3|wav` — аудио, голосовые
- любые дополнительные документы (`*.pdf`, `*.txt`) из входных сообщений

Если во "Входящей информации" есть раздел "Медиафайлы:", используй его как чек-лист:

```
Медиафайлы:
лежат в media/
img_1234567890_abc12345_coconut.jpg
vid_1234567890_demo.mp4
doc_1234567890_specs.pdf
```

### 1. Общие правила
1. **Всегда обрабатывай КАЖДЫЙ файл из списка.** Нельзя игнорировать медиу, даже если она кажется второстепенной.
2. **Пути только относительные**:
   - Из корня KB: `media/...`
   - Из `topics/`: `../media/...`
   - Из `topics/subfolder/`: `../../media/...`
3. **Не добавляй `<!-- TODO: Broken image path -->`** — пути уже проверены.
4. **Не дублируй вставки**: одно и то же медиа в одном файле вставляется один раз, но можно использовать его в разных файлах.

### 2. Изображения (`img_*.jpg|png`)

1. **Вставка** (пример для `topics/`):
   ```markdown
   ![Краткое описание](../media/img_123456_slug.jpg)

   **Описание:** Что изображено, какие элементы важны, выводы.
   ```
2. **Два описания** обязательны: alt-text + текст под изображением (GitHub не показывает alt).
3. **Читай метаданные**:
   - `img_123_slug.md` — OCR и заметки, путь совпадает с изображением (`../media/img_123_slug.md`).
   - Используй OCR для точных формулировок и цитат.
4. **Логика размещения**: рядом с текстом, который иллюстрирует изображение, после заголовка, не в самом начале файла.

### 3. Видео, аудио, документы

Эти типы нельзя встроить как изображение. Используй ссылку + текстовый комментарий:

```markdown
**Видео:** [Демо работы сервиса](../media/vid_123_demo.mp4)
Кратко опиши, что видно/слышно и какие выводы нужно сделать.

**Аудио:** [Интервью с командой](../media/aud_456_interview.mp3) – перечисли ключевые тезисы.

**PDF/Документ:** [Техническая спецификация](../media/doc_789_specs.pdf) – перечисли, что именно важно извлечь из файла.
```

Правила:
- На каждую ссылку добавляй пояснение **перед или сразу после** неё.
- Если документ длинный, выдели основные разделы/факты, не переписывай файл целиком.
- Для видеороликов зафиксируй тайм-коды, если они важны.

### 4. Используй все файлы

- Сверься со списком `Медиафайлы:` и убедись, что каждое имя использовано либо через `![]()` (для изображений), либо через Markdown-ссылку (для остальных типов).
- Если медиа не подходит для прямой вставки, добавь текстовый вывод из него (что было на скрине, что говорилось в аудио и т.д.).
- Нельзя игнорировать вложения только потому, что они «неудобные».

### 5. Краткий чек-лист перед завершением
1. Все пути начинаются с `media/`, `../media/` или `../../media/`.
2. Каждое изображение имеет alt-text + описание под ним.
3. На каждое не-графическое медиа есть кликабельная ссылка и пояснение.
4. Нет дубликатов и пустых ссылок.
5. Информация из `.md`-метаданных учтена (для изображений).

Следуя этим правилам, ты обеспечиваешь единый стандарт работы с медиа и сохраняешь контекст для команды.
